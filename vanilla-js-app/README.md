## üìä Architecture

```
User Interaction
      ‚Üì
Event Listeners (app.js)
      ‚Üì
State Management (AppState object)
      ‚Üì
View Rendering (render functions)
      ‚Üì
DOM Updates (smooth animations)
```

**Key Principles:**
- **Separation of concerns** - Data, logic, and presentation separate
- **Immutable state** - State changes trigger re-renders
- **Event-driven** - All interactions handled by event listeners
- **Declarative rendering** - HTML is generated from state
